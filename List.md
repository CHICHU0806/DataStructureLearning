
# ⭐ 各种线性表结构的区别（顺序表、单链表、双向链表、循环链表、双向循环链表）
下面从 **结构、存储方式、优缺点、适用场景** 进行对比。

---

# 1️⃣ 顺序表（Array List / AList）

### 📌 结构

* 用 **一段连续的内存** 存储元素。
* 元素之间物理位置相邻。

### 📌 优点

* **随机访问能力强**（O(1) 时间访问第 k 个元素）
* 内存利用高、结构简单
* 遍历非常快（缓存友好）

### 📌 缺点

* 插入和删除平均需要移动大量元素，效率低（O(n)）
* 容量固定，需要预先分配空间或扩容
* 内存必须连续，可能导致分配失败

### 📌 适用场景

* 读操作远多于写操作。
* 数据大小变化不频繁。
* 要求能快速按下标访问。

---

# 2️⃣ 单链表（Single Linked List）

### 📌 结构

* 每个节点包含：

    * data
    * next 指针
* 节点可分散在内存各处。

### 📌 优点

* 插入、删除只需改变指针，时间为 O(1)
* 内存动态分配，灵活，不需要连续空间

### 📌 缺点

* 无法随机访问，只能从头遍历（O(n)）
* 只能单方向遍历

### 📌 适用场景

* 插入和删除频繁
* 不需要按位置随机访问

---

# 3️⃣ 双向链表（Doubly Linked List）

### 📌 结构

每个节点包含：

* prev 指针
* data
* next 指针

### 📌 优点

* 可双向遍历
* 删除任意节点 O(1)
* 插入某位置 O(1)

### 📌 缺点

* 每个节点多了一个 prev 指针，占更多内存
* 实现比单链表复杂

### 📌 适用场景

* 需要双向遍历
* 插入与删除操作非常频繁

---

# 4️⃣ 单向循环链表（Circular Linked List）

### 📌 结构

* 最后一个节点指向头节点
* 形成一个环

### 📌 优点

* 从任意位置开始都可以把链表当成从头遍历
* 适合循环使用的数据结构

### 📌 缺点

* 处理结束条件需要小心（不能用 NULL 判断终点）
* 只支持单方向操作

### 📌 适用场景

* 逻辑上需要循环结构，例如：

    * 约瑟夫问题
    * 循环队列
    * 游戏中的轮流机制

---

# 5️⃣ 双向循环链表（Doubly Circular Linked List）

### 📌 结构

* 双向链表 + 循环
* 头、尾节点互相连接

### 📌 优点

* 支持双向移动
* 无头无尾限制，结构完全循环
* 任意节点删除/插入依然 O(1)

### 📌 缺点

* 指针最多，需要更多内存
* 实现稍复杂

### 📌 适用场景

* 需要频繁地按前驱/后继移动
* 逻辑上需要循环结构

---

# 🌟 一张表总结所有区别

| 数据结构   | 存储方式 | 是否连续 | 是否可双向 | 是否循环 | 访问第 k 个 | 插入删除 | 内存开销 |
| ------ | ---- | ---- | ----- | ---- | ------- | ---- | ---- |
| 顺序表    | 数组   | ✔    | ✘     | ✘    | O(1)    | O(n) | 低    |
| 单链表    | 链式节点 | ✘    | ✘     | ✘    | O(n)    | O(1) | 中    |
| 双向链表   | 链式节点 | ✘    | ✔     | ✘    | O(n)    | O(1) | 较高   |
| 单向循环链表 | 链式节点 | ✘    | ✘     | ✔    | O(n)    | O(1) | 中    |
| 双向循环链表 | 链式节点 | ✘    | ✔     | ✔    | O(n)    | O(1) | 高    |
