# ⭐ 各种线性表结构的区别

（顺序表、单链表、双向链表、循环链表、双向循环链表、静态链表）

从 **结构、存储方式、优缺点、适用场景** 进行系统对比。

---

# 1️⃣ 顺序表（Array List / AList）

### 📌 结构

* 使用 **连续内存** 存储元素。

### 📌 优点

* **随机访问 O(1)**
* 遍历快（缓存友好）
* 实现简单

### 📌 缺点

* 插入删除要移动大量元素 O(n)
* 内存必须连续
* 必须预分配或扩容

### 📌 适用场景

* 读多、写少的场景
* 需要快速按下标访问

---

# 2️⃣ 单链表（Single Linked List）

### 📌 结构

* 每个节点含 `data + next`

### 📌 优点

* 插入、删除 O(1)
* 不需要连续内存
* 动态扩张灵活

### 📌 缺点

* 随机访问 O(n)
* 只能单向遍历

### 📌 适用场景

* 插入删除多
* 不需要按位置随机访问

---

# 3️⃣ 双向链表（Doubly Linked List）

### 📌 结构

* 每节点含 `prev + data + next`

### 📌 优点

* 双向遍历
* 任意节点删除 O(1)

### 📌 缺点

* 额外占用 prev 空间
* 实现复杂度更高

### 📌 适用场景

* 双向遍历
* 修改频繁

---

# 4️⃣ 单向循环链表（Circular Linked List）

### 📌 结构

* 尾节点指向头节点，形成环

### 📌 优点

* 任意位置都可作为起点遍历
* 适合循环逻辑

### 📌 缺点

* 结束条件不再用 NULL
* 只能单方向

### 📌 适用场景

* 游戏轮询
* 约瑟夫问题
* 循环队列逻辑

---

# 5️⃣ 双向循环链表（Doubly Circular Linked List）

### 📌 结构

* 双向链表 + 尾首相连

### 📌 优点

* 完全循环
* 任意节点插删 O(1)
* 双向移动方便

### 📌 缺点

* 最多的指针数量
* 内存占用最高

### 📌 适用场景

* 双向 + 循环逻辑
* 插删非常频繁

---

# 6️⃣ 静态链表（Static Linked List / Cursor List）

### 📌 结构

* 用 **数组模拟链表**
* 每个元素是结构体：

```
data
next   // 下一个元素的下标
```

* `next = -1` 相当于 NULL。

### 📌 优点

* 不需要指针，适合不支持指针的场景（早期 C 语言教材、系统设计）
* 物理结构是数组，分配在连续空间
  → 比动态链表略快，缓存友好
* 插入/删除 O(1)

### 📌 缺点

* 大小必须预先分配，不能自动扩容
* 管理“备用链表”（free list）比较复杂
* 用数组模拟链，不如真链表灵活

### 📌 适用场景

* **内存空间有限/受限的平台（如嵌入式）**
* **不允许动态分配（禁用 malloc/new）**
* **需要链表操作但希望避免指针风险的环境**

➡ 一个典型例子：
在一些 MCU（单片机）项目中，堆 malloc 可能被禁用，这时候就只能用静态链表。

# ⭐ 总结
| 数据结构       | 适用场景总结                       |
| ---------- | ---------------------------- |
| 顺序表        | 读多写少、随机访问频繁               |
| 单链表        | 插入删除多、不需要随机访问               |
| 双向链表       | 需要双向遍历、频繁修改                 |
| 单向循环链表     | 需要循环遍历、游戏轮询等               |
| 双向循环链表     | 需要双向循环遍历、频繁插删               |
| 静态链表（游标链表） | 内存受限、禁用动态分配、需要链表操作但避免指针风险 |
